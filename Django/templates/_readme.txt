루트 경로에서 사용하는 HTML 템플릿이 정의된 디렉토리입니다.
404, 500 등의 오류 페이지와 기본 페이지들이 정의되어 있습니다.

작성방법:
1. base 디렉토리에서 HTML의 기본 구조를 상속받아 작성합니다. {% extends 'base/파일명.html' %}로 상속받아 사용합니다.
2. 기타 필요한 요소가 있다면 parts에서 작성한 HTML을 include하여 사용합니다. {% include 'parts/파일명.html' %}
3. 재사용 가능한 요소가 아닐경우, 주석과 함께 해당 파일 내에 작성합니다.

기타 궁금한 내용이 있다면 라이브 커뮤니티에 질문해주세요. 감사합니다.

필요 페이지:

/
메인 페이지. 광고 배너가 표시되며, 여행지 게시글을 확인할 수 있습니다.

/signup
회원가입 페이지. 사용자가 회원가입을 할 수 있는 페이지입니다.
일반 사용자, 여형 회원, 파트너 계정으로 가입 이렇게 3개의 탭이 있으며, 각 탭에 맞는 회원가입 폼을 작성할 수 있습니다.
여성회원 및 파트너 계정은 가입 시 관리자의 승인이 필요합니다.
각 계정별로 필요한 정보는 해당 앱 및 사이트 맵 문서를 참고해주세요.

/find_account
계정 찾기 페이지. 사용자가 계정을 찾을 수 있는 페이지입니다.
* 수정됨. 계정 찾기 기능은 존재하지 않습니다. 해당 페이지에 관리자에게 문의 버튼을 통해 쪽지 페이지로 이동하도록 수정되었습니다.(비회원도 관리자에게 쪽지 발송 가능)

/profile
프로필 페이지. 사용자의 프로필 정보를 확인하고 수정할 수 있는 페이지입니다.
관리자 계정의 경우, 다른 사용자의 프로필 정보를 확인하고 수정할 수 있습니다.(사용자 관리 권한 필요)

/activity
활동 페이지. 사용자의 활동 내역을 확인할 수 있는 페이지입니다.
각 활동별로 활동 메세지, 포인트 변동 내역, 날짜를 확인할 수 있습니다.
관리자 계정의 경우, 다른 사용자의 활동 내역을 확인할 수 있습니다.(사용자 관리 권한 필요)

/terms
이용약관 페이지. 서비스 이용약관을 확인할 수 있는 페이지입니다.
이용약관 본문은 서버로부터 받아와서 표시됩니다. 관리자가 수정 가능.

/bookmark
북마크 페이지. 사용자가 북마크한 게시글을 확인할 수 있는 페이지입니다.
페이징 기능은 없습니다.(모두 한 페이지에 표시)

/404
404 오류 페이지입니다. 페이지를 찾을 수 없을 때 표시됩니다.

필요한 요소 및 contexts 정보는 해당 앱 및 사이트 맵 문서를 참고해주세요.
